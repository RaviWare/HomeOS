# HomeOS user personas — review pack

Use this with **Onboarding (how you use HomeOS)**, **Vault Settings → role**, and **Command Deck → Customize**.

## First-person principle

You never introduce yourself as “Tenant.” That’s what an owner might call you.
HomeOS asks **how you use the product** in the first person:

- **I rent my home** (storage id still `Tenant` for compat)
- **I own & live here** (`Homeowner`)
- **I rent out homes** (`Landlord`)
- etc.

## Selection flow

1. **Category** (Individuals first): Individuals · I rent out/invest · I operate many homes · I advise · I work on-site  
2. **Exact role** inside that category (first-person titles)  
3. Optional **custom role** title (“I manage my parents’ flat”)  
4. HomeOS seeds Command Deck: default Life/Trends/Datasets, nav priority, ribbon  
5. User can **rearrange / hide / pin modules** anytime (Customize)

| | |
|--|--|

| | |
|--|--|

## Customization (per user, on-device)

In **Command Deck → Export · Customize**:

1. **Dashboard persona** — Auto from role, or force any of the 5  
2. **Show secondary modules** — de-emphasized modules visible or not  
3. **Hide modules** — personal (Settings always kept)  
4. **Pin modules** — float favorites  
5. **Default deck view** — Life OS / Trends / Datasets  
6. **Weather · motion · counts**  

After persona change: reopen the sidebar once so nav refreshes.
